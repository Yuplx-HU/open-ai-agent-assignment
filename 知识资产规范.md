# 知识资产规范

## 1. 知识集定义

**知识集**指一个逻辑上自洽的知识整体——如一篇学术论文、一本教材、一门课程、一段百科条目等。

**独立知识集的界定标准**（满足任一即可）：
- 有唯一标识符（DOI、URL、版本号），可独立访问
- 有独立的创作时间和作者 / 发布者
- 内容自洽，不依赖上下文即可理解

---

## 2. 来源类型

| 类型 | 文件夹 | 允许格式 | 来源举例 |
| --- | --- | --- | --- |
| 学术论文 | `01-academic/` | PDF、MD、TXT | Nature、Science、ScienceDirect |
| 学科教材 | `02-textbook/` | PDF、MD、TXT、EPUB | 正式出版教材 |
| 课程资源 | `03-course/` | PDF、PPT、MD、Word、TXT | MIT OCW、MOOC |
| 百科资源 | `04-encyclopedia/` | PDF、MD、TXT、HTML | Wikipedia、百度百科 |
| 项目工程文件 | `05-project/` | 不限（不含流数据） | GitHub、Hugging Face |
| 领域知识库 | `06-knowledge-base/` | 不限（不含流数据） | UniProt、Gene Ontology、PDB |
| 技术博客 | `07-blog/` | PDF、MD、TXT、HTML | Medium、知乎 |
| 领域论坛 | `08-forum/` | PDF、MD、TXT、HTML | Reddit、Stack Overflow |

---

## 3. 存储结构

```
data/
├── 01-academic/          # 学术论文
├── 02-textbook/          # 学科教材
├── 03-course/            # 课程资源
├── 04-encyclopedia/      # 百科资源
├── 05-project/           # 项目工程文件
├── 06-knowledge-base/    # 领域知识库
├── 07-blog/              # 技术博客
└── 08-forum/             # 领域论坛
```

每个知识集在其类型文件夹下以 `{knowledge_set_id}/` 为单元组织：

```
{knowledge_set_id}/
├── content/              # 原始内容
├── keywords.json         # 主题与关键词标注
└── source.json           # 来源元数据
```

---

## 4. 标注文件格式

### 4.1 keywords.json

主题和关键词全部使用 **kebab-case** 格式。

```json
{
  "protein-corona": ["protein-nanoparticle-interaction", "metal-based"],
  "data-mining": ["feature-selection", "classification"]
}
```

### 4.2 source.json — 学术论文

```json
{
  "doi": "10.18653/v1/2026.findings-eacl.62",
  "title": "HiGraAgent: Dual-Agent Adaptive Reasoning ...",
  "journal": "Findings of EACL 2026",
  "authors": ["Luu, Hung", "Nguyen, Long S. T."],
  "url": "https://doi.org/10.18653/v1/2026.findings-eacl.62",
  "publication_date": "2026"
}
```

### 4.3 source.json — 其他网络资源

```json
{
  "url": "https://example.com/article",
  "date": "2026-05-21"
}
```

---

## 5. 采集注意事项

- 除可公开 API 访问的资源外，原始文件**必须保存到本地**
- 网页抓取时去除无关内容（广告、导航栏等）
- 领域论坛的问答数据必须**完整保留双方问答轨迹**，不得节选
- 所有知识集**必须纳入知识库**，且智能体可以检索到
- 如果搜集了"项目工程文件"类型的知识集，则必须把该项目构建为 MCP 工具或 SKILL

---

## 6. 数量要求

每组须搜集并整理**至少 20 个有效知识集**。每个知识集须同时包含 `content/`、`keywords.json`、`source.json` 三个文件，缺一不可。

> 有效知识集指内容充实、来源可查、符合规范的知识集。重复提交相同内容、不同ID指向同一来源、或缺少任一必需文件的，不计入有效数量。
