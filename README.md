# AI Agent 课程开放式项目作业

## 一、作业概述

本作业为**小组**开放式课程项目,学生根据小组的**课题方向**自由选择项目侧重点,将 AI Agent 技术与自身研究方向相结合。

**推荐使用现有的 Agent 框架**(如 LangChain、CrewAI、Agno、AutoGen、Dify 等)完成作业,以体验当前的 Agent 技术生态。不限制框架选择,各组可根据自身需求和熟悉程度自行决定。

小组需在以下两个方向中自主选择**侧重点并分配权重**:

| 方向 | 内容 | 说明 |
|------|------|------|
| **方向A** | 知识资产收集 + 评测基准构建 | 注重知识库建设与评测 |
| **方向B** | 智能体应用项目 | 注重 Agent 在自己课题方向的应用 |

小组自主设定两个方向的**权重比例**(两者之和 = 100%),分别完成对应内容。

## 二、方向A:知识资产与评测基准

### 2.1 知识资产

围绕自己的研究方向,收集并整理高质量知识集,构建结构化知识库。

- 至少收集 **20 个有效知识集**
- 每个知识集须包含三个文件:`content/`、`keywords.json`、`source.json`
- 覆盖至少 **3 种来源类型**(学术论文、教材、课程资源等)
- 详细的收集规范、存储结构、标注格式见 **[知识资产规范.md](知识资产规范.md)**

### 2.2 评测基准

基于所收集的知识资产,构建一套可用于评测 Agent 问答能力的标准化问题集。

- 至少 **20 个有效问题**
- 问题格式为 JSON,包含完整元数据
- 答案准确,来源引用正确
- 详细的问题格式、答案结构、标注规范见 **[评测基准规范.md](评测基准规范.md)**

### 2.3 方向A评分标准(满分100分)

| 评分项 | 满分 | 评分规则 |
|--------|------|---------|
| **知识资产 - 数量** | 15 | ≥20个 = 15分;15-19个 = 10分;<15个 = 5分 |
| **知识资产 - 完整性** | 10 | 三个文件齐全,无缺失 = 10分;部分缺失 = 5分 |
| **知识资产 - 格式规范** | 10 | 全部使用kebab-case,无格式错误 = 10分;部分错误 = 5分 |
| **知识资产 - 来源广度** | 10 | 覆盖≥3种来源类型 = 10分;2种 = 6分;1种 = 3分 |
| **知识资产 - 质量** | 10 | 内容充实、来源可靠 = 10分;基本合格 = 6分 |
| **评测基准 - 数量** | 15 | ≥20个 = 15分;15-19个 = 10分;<15个 = 5分 |
| **评测基准 - 答案准确** | 10 | 全部正确 = 10分;有部分错误 = 5分 |
| **评测基准 - 格式规范** | 10 | JSON格式正确,字段齐全 = 10分;部分错误 = 5分 |
| **评测基准 - 问题质量** | 10 | 问题清晰、有来源引用 = 10分;基本合理 = 6分 |

## 三、方向B:智能体应用项目

> 本部分为**开放式项目**,不规定具体模块内容。智能体可用于知识问答、自动化流水线、数据分析、实验辅助等多种场景,学生结合自己的课题方向自由设计。

### 3.1 项目要求

**核心原则**:项目必须与学生的**课题方向**相结合,体现 AI Agent 技术在该方向上的应用价值。

**项目形式(包括但不限于)**:
- 基于 Agent 的知识问答系统
- 科研自动化流水线(实验设计、数据分析、结果解读)
- 文献检索与综述生成 Agent
- 实验数据处理与分析 Agent
- 多 Agent 协作完成复杂科研任务
- 其他与课题方向结合的 Agent 应用

**提交要求**:
1. **项目提案**(Proposal):在项目开始前提交,说明项目目标、技术方案、预期成果
2. **最终报告**(Final Report):项目完成后提交,包含系统设计、实现细节、结果分析
3. **代码与演示**:可运行的系统代码,以及演示视频或截图

### 3.2 方向B评分标准(满分100分)

| 评分维度 | 满分 | 评分规则 |
|----------|------|---------|
| **项目定义与规划** | 20 | 提案清晰,目标明确,技术方案合理 = 20分;基本完整 = 12分;明显缺陷 = 5分 |
| **功能完成度** | 30 | 核心功能全部实现,系统可正常运行 = 30分;部分功能未完成 = 18分;仅有原型 = 8分 |
| **技术实现质量** | 20 | 代码质量高,模块清晰,有错误处理 = 20分;基本可运行 = 12分;代码混乱 = 5分 |
| **与课题方向结合** | 20 | 紧密结合课题方向,有明确应用价值 = 20分;基本相关 = 12分;关联性弱 = 5分 |
| **文档与展示** | 10 | 文档完整,演示清晰 = 10分;基本完整 = 6分;文档缺失 = 2分 |

**功能完成度说明**:
- 知识问答类项目:须能正确回答领域相关问题,检索机制有效
- 自动化流水线类项目:须能完成端到端的自动化处理
- 数据分析类项目:须能正确执行分析并输出结果
- 其他类型项目:须实现核心功能并展示效果

## 四、权重分配与总分计算

### 4.1 权重分配规则

学生自主设定方向A和方向B的权重,须满足:
- **Weight_A + Weight_B = 100%**
- 建议每部分权重不低于 **30%**(避免某一部分过于单薄)

**示例权重分配**:

| 学生类型 | 方向A权重 | 方向B权重 | 适用场景 |
|---------|-----------|-----------|---------|
| 理论型 | 70% | 30% | 注重知识库建设,项目作为补充 |
| 应用型 | 30% | 70% | 注重Agent应用,知识库作为支撑 |
| 均衡型 | 50% | 50% | 两方面均衡投入 |

### 4.2 总分计算

```
总分 = 方向A得分 × Weight_A + 方向B得分 × Weight_B
```

**示例**:Weight_A = 60%,Weight_B = 40%,方向A = 85分,方向B = 90分 → 总分 = 85×0.6 + 90×0.4 = **87分**

## 六、常见问题

**Q1:小组可以两个方向都做满吗?**
A:可以,但权重之和必须为100%。例如A=50%,B=50%。

**Q2:方向B的项目可以用方向A的知识库吗?**
A:可以,且鼓励这样做。方向A的知识库可以为方向B的项目提供支撑。

**Q3:如果小组课题方向不涉及编程,如何完成方向B?**
A:方向B不一定是纯编程项目,可以是Agent工具的使用与评估、工作流程设计、提示词工程等。

**Q4:方向A的知识资产和评测基准有数量上限吗?**
A:无上限,越多越好。评分标准中的"至少20个"是最低要求。

**Q5:必须使用推荐的Agent框架吗?**
A:不强制,但强烈推荐。使用现有框架可以更好地体验当前Agent技术的核心能力(工具调用、记忆管理、多Agent协作等),也是本课程的重要学习目标。

**Q6:小组成员如何分工?**
A:分工由小组自行决定,但建议明确各成员的职责并在最终报告中说明。常见分工方式:按方向分工(部分成员负责方向A,部分负责方向B)、按角色分工(知识采集、系统开发、文档撰写等)。
